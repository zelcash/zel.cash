What is Flux?

“An operational network of decentralized computing power is one of the core modules to the Zel Ecosystem, providing the ability for Zel and 3rd party developers alike to run their applications on a fair-priced network with no single point-of-failure” - Daniel Keller COO

Zel has access to a decentralized computer network which is connected to an operating system we call Flux.

The decentralized computational network is stood up by individual Zel holders in running Zelnodes.

These Zelnodes are geographically-spaced all over the world, so developers can be assured that their dockerized apps are accessible by users at all times and locations.

Flux allows the deployment and management of any dockerized app around the world via API and UI.

How does it work?

Spawning an application works in four easy steps:

1. Visit a Flux via your web browser (preferably google chrome). From there you need to log in to the Flux to proceed with the application registration process. You can do this by either signing a message with your ZelID in Zelcore or by signing a message with a btc address itself Flux instances to connect to:

  • http://193.188.15.253:16126

  • http://188.239.61.122:16126/

  • http://144.91.105.147:16126/

  • http://5.189.179.96:16126/

    • 2. Register an application on any flux node via filling a registration form. If you filled in everything correctly and passed the validation checks you will see an application hash, zel-address and an amount of how much you need to pay for one month of running your personal application.

    3. Perform the transaction on the Zel blockchain:

    Send the stated Zel amount from step 1. and the application hash as a message in the op_return field*
    * Important! You only have a 1 hour interval (from the initial registration in step 1) in which this transaction has to be mined on the network. If it's not mined in time, the app data is lost and new registration and payment has to be made.

    4. Flux will do its magic and spawn your application Here is an in depth step by step guide by our beloved Goose, check it out if you need more guidance.

    Flux Pricing Models

    With Flux we want to give Docker app builders the ability to deploy the application onto our decentralized computational network at much lower cost than renting VPS’s/dedicated servers.” - Parker Honeyman CTO

    Our simple introductory pricing models are straightforward and easy to calculate and from 5 zel per instance.

    CPU: 3 ZEL per 0.1 vCPU core

    RAM: 1 ZEL per 100mb

    HDD: 0.5 ZEL per 1GB
